# knowledge_base
База знаний, предназначенная для хранения и обработки данных о произошедших авариях на ПО

## Запуск инфраструктуры
1) Включить Docker Desktop

2) В папке .\lakehouse_infra в терминале выполнить команду:
docker-compose up -d

3) Готово. То что кафка падает - это нормально

В Docker Desktop можно перейти на сервис Trino для просмотра статусов выполнения запросов
В Docker Desktop можно перейти на сервис MinIO для просмотра состояния хранилища данных (данные записались или нет)

## Запуск панели администратора
1) Запустить инфраструктуру

2) В папке .\admin_panel в терминале выполнить команду python .\main.py

3) Готово

Для загрузки тестовых данных запустить скрипт в папке .\admin_panel\scripts
python .\load_synthetic_to_lakehouse.py

## Подключение к Trino через Dbeaver
1) Создать соединение Trino

2) Внести такие настройки
connected by: Host
Host: localhost port: 8082
DB/Schema: lakehouse

user: admin
password: *оставить пустым*
